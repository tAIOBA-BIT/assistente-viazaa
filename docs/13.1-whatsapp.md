# 💬 Central de Comunicação via WhatsApp

> **Módulo:** Central de Comunicação via WhatsApp  
> **Documento:** 09  
> **Versão:** 1.0.0

---

# 📖 Objetivo

Este módulo é responsável por realizar automaticamente toda a comunicação referente às retarificações através do WhatsApp.

Seu objetivo é garantir que todas as informações sejam enviadas de forma rápida, padronizada e segura, reduzindo o tempo operacional e eliminando erros de comunicação.

O envio ocorrerá somente após a confirmação do operador na Central de Conferência.

---

# 🎯 Objetivos

O módulo deverá ser capaz de:

- Localizar automaticamente o grupo correto;
- Identificar o comercial responsável;
- Montar automaticamente a mensagem;
- Identificar o trecho retarifado;
- Mencionar o comercial responsável;
- Enviar uma mensagem para a gestora do setor;
- Registrar o resultado do envio;
- Informar falhas ao operador.

---

# 🏗️ Fluxo Geral

```text
Central de Conferência

        │

        ▼

Operador confirma

        │

        ▼

Buscar grupo

        │

        ▼

Montar mensagem

        │

        ▼

Mencionar comercial

        │

        ▼

Enviar mensagem

        │

        ▼

Registrar resultado
```

---

# 📥 Informações Recebidas

O módulo receberá:

- Número da OP;
- Número do Pedido;
- Passageiro;
- Valores pesquisados;
- Melhor tarifa;
- Companhia aérea;
- Trecho retarifado;
- Comercial responsável;
- Grupo do WhatsApp.

---

# 👥 Localização do Grupo

O grupo será localizado automaticamente através das regras cadastradas.

Exemplo:

```
LATAM

↓

Grupo LATAM
```

ou

```
Empresa

↓

Grupo Corporativo
```

Caso nenhum grupo seja encontrado, o operador poderá selecionar manualmente.

---

# 👤 Comercial Responsável

O sistema localizará automaticamente o comercial associado ao pedido.

Durante a montagem da mensagem, esse usuário será mencionado.

Exemplo:

```
@João Silva
```

Caso o WhatsApp utilizado permita marcação por ID, o sistema utilizará o identificador correspondente.

---

# 📝 Estrutura da Mensagem

A mensagem seguirá um padrão único.

Exemplo:

```text
Pedido 908082 / VZ11988

Wooba: R$ 1.495,89
Kontrip: R$ 1.362,42
LATAM: R$ 958,80

Valor Total do Pedido: R$ 1.417,73

Trecho LATAM retarifado.

@Comercial Responsável
```

---

# ✈️ Identificação do Trecho

O sistema deverá informar claramente qual trecho sofreu alteração.

Exemplos:

```
Trecho de ida retarifado.
```

```
Trecho de volta retarifado.
```

```
Ambos os trechos foram retarifados.
```

Caso o trecho não possa ser identificado automaticamente, será exibido:

```
Trecho não identificado.
Confirmar manualmente.
```

---

# 💰 Informações Financeiras

Sempre que houver pesquisa de tarifas, deverão ser exibidos:

- Valor encontrado em cada portal;
- Valor original do pedido;
- Melhor tarifa encontrada.

O menor valor deverá ser destacado na interface da Central de Conferência.

---

# 📤 Envio

Após a confirmação do operador:

- Localizar grupo;
- Abrir conversa;
- Inserir mensagem;
- Mencionar comercial;
- Confirmar envio;
- Registrar horário.

---

# 👩‍💼 Comunicação com a Gestora

Após o envio ao grupo operacional, o sistema poderá enviar uma segunda mensagem para a gestora do setor.

Essa mensagem terá como objetivo apenas registrar que houve uma retarificação tratada pelo aplicativo.

Exemplo:

```text
Retarificação processada.

Pedido: 908082

Operador: Carlos

Melhor tarifa localizada.

Comunicação enviada ao grupo correspondente.
```

Essa funcionalidade poderá ser habilitada ou desabilitada nas configurações.

---

# ⚠️ Tratamento de Erros

## Grupo não encontrado

Permitir seleção manual.

Registrar ocorrência.

---

## Comercial não localizado

Enviar a mensagem sem menção.

Registrar a inconsistência.

---

## WhatsApp indisponível

Informar ao operador.

Permitir nova tentativa.

---

## Erro no envio

Registrar:

- Data;
- Hora;
- Motivo;
- Mensagem;
- Grupo.

---

# 📋 Registro

Cada envio deverá gerar um registro contendo:

- OP;
- Pedido;
- Grupo;
- Comercial;
- Data;
- Hora;
- Operador;
- Status do envio.

---

# 🔄 Reenvio

O operador poderá reenviar a última mensagem.

O sistema utilizará exatamente o mesmo conteúdo anteriormente aprovado.

---

# 📊 Histórico

Todas as mensagens enviadas ficarão registradas.

Será possível consultar:

- Data;
- Pedido;
- OP;
- Grupo;
- Comercial;
- Conteúdo enviado;
- Operador responsável.

---

# 🛡️ Segurança

O módulo nunca enviará mensagens automaticamente sem confirmação do operador.

Toda comunicação será registrada para fins de auditoria.

---

# 🚀 Melhorias Futuras

O módulo poderá evoluir para incluir:

- Envio do PDF diretamente no grupo;
- Confirmação de leitura (quando suportado);

# 🧩 Integração

Recebe informações de:

Envia informações para:

- Registro;
- Dashboard;
- Histórico.

# 📌 Observações para Desenvolvimento

A montagem da mensagem deverá utilizar templates configuráveis.

Nenhum texto deverá ficar fixo no código.

Isso permitirá que a empresa altere o padrão das mensagens sem necessidade de atualizar o aplicativo.

---

# 💡 Templates de Mensagens

O sistema deverá possuir um gerenciador de modelos de mensagens.

Exemplo:

## Modelo Padrão

```text
Pedido {pedido} / {op}

Wooba: {wooba}

Kontrip: {kontrip}

LATAM: {latam}

Valor Total do Pedido: {valor_total}

{trecho}

@{comercial}
```

As variáveis serão substituídas automaticamente durante o envio.

Novos modelos poderão ser criados para diferentes tipos de operação, clientes ou setores.
